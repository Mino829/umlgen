# umlgen無料導入サポート

最初の**10リポジトリ**を対象に、Javaプロジェクトへumlgenを無料で導入します。

目的は売上ではなく、実際のPull Requestで継続利用される条件と、導入時につまずく点を確認することです。CLIとGitHub Actions workflowはサポート終了後も無料で利用できます。

## 提供する内容

- 初回30分の導入支援
- umlgenのGitHub Actions workflow設定
- Javaソースパス、除外パス、関係の深さの初期設定
- テスト用Pull RequestでPlantUML／SVG artifactが生成されることの確認
- 導入後の軽微な設定調整1回

設定は原則として、利用者が確認してマージできるPull Requestまたはコピペ可能なworkflowとして提供します。

## お願いすること

- 導入から1〜2週間後の短いフィードバック
- 導入にかかった時間と、生成できなかったケースの共有
- 継続して利用するか、利用しない場合はその理由の共有

公開事例や好意的な感想は参加条件ではありません。率直な結果をプロダクト改善へ利用します。

## 対象

- GitHubで管理しているJavaプロジェクト
- GitHub Actionsを利用できる
- Maven、Gradle、または一般的な複数モジュール構成
- 公開可能な範囲で技術構成を共有できる

Public／Privateどちらでも申し込めます。GitHub Enterprise Server、特殊なセルフホストランナー、独自CIへの移植は無料サポートの対象外となる場合があります。

## 申し込みから導入まで

1. [無料導入Issue](https://github.com/Mino829/umlgen/issues/new?template=onboarding.yml)を作成する
2. 公開情報だけで構成と実現したいことを確認する
3. 対象条件と残り枠を確認する
4. GitHub Actionsを設定する
5. テスト用Pull Requestでartifactを確認する
6. 1〜2週間後に利用結果を共有する

料金は発生しません。10リポジトリへ到達した後は募集を停止または条件を変更する場合がありますが、すでに開始した無料サポートを途中から有料にすることはありません。

## セキュリティ

申し込みIssueは公開されます。次の情報は書かないでください。

- private repositoryの名前、URL、ソースコード
- アクセストークン、APIキー、secret
- 社外秘の組織名や顧客情報
- 個人情報

Privateの場合は「Java 21／Spring Boot／Gradle／3モジュール」のように、公開可能な技術構成だけを記載してください。ソースコードを外部の図生成サービスへ送信する構成にはしません。

## サポート範囲

無料サポートには、初回導入と設定調整1回を含みます。新しい言語対応、umlgen本体の新機能開発、他のCIサービスへの移植、継続的な保守作業は含みません。

自分で導入する場合は[GitHub Actions導入ガイド](github-actions.md)を利用できます。
