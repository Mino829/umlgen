# umlgen無料デザインパートナー募集

Java／SpringプロジェクトでPull Requestレビューをしている**3チーム**を募集しています。

umlgenを無料で導入し、PRごとに「どのクラスと依存関係が変わったか」を図で確認できる状態を作ります。4週間使っていただき、役に立った場面と使いにくかった場面を教えてください。

[30秒デモを見る](../examples/order-service/README.md)

## 提供するもの

- GitHub Actionsへの無料導入
- Javaソースパス、除外、関係の深さの初期設定
- Pull Request差分図の動作確認
- 4週間の軽微な設定調整
- 開発者本人による導入支援

プログラム参加中の費用はかかりません。終了後もOSSのCLIとGitHub Actions workflowは無料で利用できます。プログラム終了時に契約や支払いを求めることはありません。

## お願いすること

- 初回30分程度の導入確認
- 4週間、週1回以上Pull Requestで利用
- 1週目と4週目に各20分程度のフィードバック
- 生成できなかったケースや役に立たなかったケースの共有
- 匿名化した学びをプロダクト改善へ利用することへの同意

好意的な感想や公開事例を求めるプログラムではありません。率直なフィードバックを重視します。

## 対象

- Javaを使用している個人または小規模チーム
- GitHub Pull Requestでコードレビューしている
- GitHub Actionsを利用できる
- 4週間、週1回以上Pull Requestを作成する

Public／Privateどちらのリポジトリでも応募できます。ただし、GitHub Enterprise Serverや特殊なセルフホスト環境は対象外になる場合があります。

## 応募方法

[デザインパートナー応募フォーム](https://github.com/Mino829/umlgen/issues/new?template=design-partner.yml)から、公開できる情報だけを送ってください。

応募Issueは公開されます。次の情報は書かないでください。

- private repositoryの名前、URL、ソースコード
- アクセストークン、APIキー、secret
- 顧客名、個人情報、社外秘情報

Privateの場合は「Java 21／Spring Boot／Gradle／週5 PR」のように、利用技術と規模だけを記載してください。

応募内容を確認後、対象条件と残り枠に合う方へGitHub上で返信します。応募によって参加やサポートを保証するものではありません。

## 4週間の流れ

| 時期 | 内容 |
| --- | --- |
| 導入前 | 現在のレビュー方法と課題を確認 |
| 初日 | GitHub Actionsを設定し、テストPRで図を確認 |
| 1週目 | 導入時間、生成成功率、最初の利用感を確認 |
| 2〜3週目 | 実際のPRで利用し、問題があれば設定調整 |
| 4週目 | 継続利用の意思と、なくて困る機能を確認 |

プログラムの進め方と評価基準は[検証プレイブック](design-partner-playbook.md)で公開しています。
